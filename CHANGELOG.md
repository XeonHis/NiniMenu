# 更新日志

## v0.6.0 (2026-06-05)

### 新增

- **上传图片自动压缩** — 手机上传的 PNG 截图/照片自动备份原图到 `uploads_backup/`，然后 EXIF 方向修正 + 缩放 + 转 JPG（quality=85），体积减少 70-90%
- **`internal/imaging` 包** — Go 图片处理核心（AutoOrientation + Lanczos 缩放 + 白底合成 + JPG 编码），依赖 `github.com/disintegration/imaging`
- **上传错误具体提示** — 前端上传失败时展示后端返回的具体原因（如"图片大小不能超过5MB"），替代通用"上传失败"文案
- **`getUploadErrorMessage`** — `api/client.ts` 新增工具函数，从 axios error 中提取后端 `message` 字段

### 优化

- **上传大小限制可配置** — `MAX_UPLOAD_SIZE_MB` 环境变量（默认 5MB），不再硬编码
- **Gin Multipart 内存限制** — `r.MaxMultipartMemory` 按上传大小配置动态设置，确保大文件请求到达 handler 校验
- **错误提示动态化** — 大小超限提示改为 `fmt.Sprintf("图片大小不能超过%dMB", maxMB)`，跟随配置值

### 新增环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `BACKUP_DIR` | `uploads_backup` | 上传原图备份目录 |
| `MAX_UPLOAD_SIZE_MB` | `5` | 上传大小上限 MB |
| `COMPRESS_MAX_DIM` | `1200` | 压缩最大宽高 px |
| `JPEG_QUALITY` | `85` | JPG 压缩质量 |

## v0.5.0 (2026-06-05)

### 新增

- **明日菜单偏好推荐** — `/pick/tomorrow` 接口，支持快速/清淡/辣/收藏/均衡偏好过滤 + 评分排序
- **应用名称自定义** — `/app-info` 接口，管理后台设置页可自定义应用显示名称
- **菜品图片字段** — RecipeSpec 新增 Image 字段，所有菜谱配置图片路径
- **imgtool 图片工具** — `backend/imgtool.exe`，支持白边去除（泛洪填充变透明）、自动裁剪、多尺寸图标生成（32/64/128/180/256/512）
- **语录管理页面** — 管理后台新增语录 CRUD 页面
- **记录管理页面** — 管理后台新增用餐记录查看页面
- **PWA 支持** — 新增 `manifest.json` 和多尺寸应用图标
- **成就解锁动画** — 前端成就解锁时显示 Overlay 动效
- **PageHeader 通用组件** — 子页面标题栏组件
- **成就进度展示** — 成就页显示进度条、解锁时间、自动/手动标签

### 优化

- **数据库性能** — SQLite WAL 模式 + 复合索引优化（pick/record/shopping）
- **成就系统防抖** — 批量处理成就检测，GROUP BY 统计事件
- **Dashboard 增强** — 分类分布、热门菜品、周趋势、难度分布统计
- **菜品管理** — 支持搜索、筛选、分页、批量启禁用/删除/改分类
- **菜品编辑重构** — 分区布局，多图上传、标签、排序权重
- **菜品详情** — 记录统计、多图画廊展示
- **History 页面** — 增强菜品图片/emoji 显示
- **Settings 重构** — Toggle/SettingRow 通用组件
- **Tomorrow 页面重构** — 偏好选择、目标道数、换菜/替换单道
- **应用图标更新** — 新版图标，旧图清理
- **记录接口** — 返回菜品图片 URL 和 emoji

### 修复

- `dayRatingApi.create` mood 字段改为可选

### 其他

- 新增 `.env` 加载支持
